Expand description

A storage for storing component data.

Safety

Implementors of this trait must ensure that get and get_mut are consistent and injective. In other words, for any a: Self::RawEntity, get(a) and get_mut(a) return the same value (only differing by mutability), and for any b: Self::RawEntity where a != b, get(a) must not alias get(b).

This implies the implementation is not safe if Eq and Ord are incorrectly implemented for Self::RawEntity, which is why entity::Raw is an unsafe trait that strictly requires complete equivalence and ordering. (Imagine if RawEntity is f64, and a and b are both f64::NAN; then a != b but get_mut(a) would still alias get_mut(b))

Returns an immutable iterator of slices over the storage, ordered by entity index order.

Each item yielded by the iterator is a tuple of (index, slice), where slice is the slice of components in the chunk, and index is the entity index of slice[0]. slice is always nonempty.
