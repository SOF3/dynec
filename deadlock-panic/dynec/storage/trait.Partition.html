Expand description

Borrows a slice of a storage, analogously &'t mut Storage[..].

This trait does not provide set because adding/removing items may cause rebalances in the tree implementation and result in dangling references in other partitions that are not &mut-locked.

source

fn by_ref(&mut self) -> Self::ByRef<'_>

Re-borrows the partition with reduced lifetime.

This is useful for calling into_iter_mut and split_at, which take self as receiver to preserve the lifetime.

source

fn split_out(&mut self, entity: Self::RawEntity) -> Self

Splits the partition further into two subpartitions, replacing self with the left partition.

entity must be > 0 and < partition_length, i.e. the expected key ranges of both partitions must be nonempty. (It is allowed to have a nonempty range which does not contain any existing keys)
