Module dynec::entity

source · [−]
Expand description

An entity is a single object that owns components.

Entities are reference-counted in debug mode, so they cannot be copied directly.

All components and states referencing entities must override the for_each_entity method.

All strong references to an entity must be dropped before it gets deleted.
